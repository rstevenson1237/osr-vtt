Activity Shell — Mockups

Companion to MASTER_PLAN_v2.md · R1. Static wireframe mockups of the redesigned room UI: near-fullscreen stage, four slim edge tabs, color-coded activity groups, single-color stroke icons, the always-on-top dice overlay, and the mobile single-activity mode. Circled letters key each board, dungeon-map style.

Choose Option A or B at WI-0 · palette values are placeholders for the R2 token sheet

Board 1 Legend — groups, icons, tab states

Every activity icon is a single-color stroke drawn as currentColor; the group supplies the color. Group boxes on the rail make the clusters — and therefore the app's structure — discoverable at a glance.

Board 4 Desktop Option B · floating cards — alternate

Same rails and registry; expansion behavior differs. Mini-cards float free (draggable, persist until dismissed), tools detach into a floating palette you park anywhere, and the log surfaces as transient toasts instead of a docked ticker. More canvas, more state to manage; suits a second-monitor referee.

Boards 5–7 Mobile / tablet — single-activity mode

Below 900px (or coarse pointer): no mini-cards — the bottom bar switches the whole stage. Tools live in a drag-handle bottom sheet. The dice overlay behaves identically to desktop. Group color appears as the active tab's underline.
